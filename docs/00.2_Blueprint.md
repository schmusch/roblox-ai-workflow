# Technische Architektur & Blueprint: [Spielname]

## 1. Rojo-Projektstruktur & Datei-Layout
* **`ServerScriptService/Server`**: Autoritatives Server-Skript-System (`src/server/`).
* **`ReplicatedStorage/Shared`**: Gemeinsam genutzte Skripte, Konfigurationen und Netzwerk-Remotes (`src/shared/`).
* **`StarterPlayerScripts/Client`**: Client-only Skripte für UI, Steuerung und visuelle Effekte (`src/client/`).
* **`StarterGui`**: Replizierte UI-Templates.

## 2. Netzwerk-Grenzen & Remote-Protokolle
* **RemoteEvents** (unter `ReplicatedStorage.Events`):
  * `[Name]RemoteEvent`: [Zweck / Payload-Beschreibung]
* **RemoteFunctions** (unter `ReplicatedStorage.Events`):
  * `[Name]RemoteFunction`: [Zweck / Payload-Beschreibung]

## 3. Server-Autorität & Sicherheits-Guardrails
* **Validierungsregeln**:
  * Der Server validiert jeden Client-Aufruf geometrisch (z. B. Proximity-Check, Kollisionsprüfung).
  * Der Server validiert alle Wirtschaftsbewegungen (z. B. keine Client-seitige Währungstransaktion ohne Server-Validierung).
* **Daten-Schutz**: Keine sensitiven Server-Geheimnisse oder privaten Module dürfen im client-sichtbaren Bereich (`ReplicatedStorage`) liegen.

## 4. Daten-Modell & State Persistence
* **Daten-Schema** (z. B. via ProfileService):
  ```luau
  export type PlayerSaveData = {
      Currency: number,
      Level: number,
      Inventory: { [string]: any },
  }
  ```
